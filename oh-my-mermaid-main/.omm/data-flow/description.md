Data flows through oh-my-mermaid in three main paths: (1) Write path — AI assistants or users invoke `omm write` which passes through the store engine to persist field files and metadata in .omm/. (2) Read/analysis path — validator, diff engine, and ref tracker read diagrams from the store to analyze content. (3) Sync path — push uploads .omm/ to the cloud API with credentials, pull downloads back with path-safe resolution, and the web viewer reads via REST + SSE live-reload.
